Introduction

Hack23 AB is committed to ensuring digital accessibility for people with disabilities. We continually improve the user experience for everyone and apply relevant accessibility standards.

Conformance Status

The Web Content Accessibility Guidelines (WCAG) defines requirements for designers and developers to improve accessibility for people with disabilities. It defines three levels of conformance: Level A, Level AA, and Level AAA.

Hack23.com is partially conformant with WCAG 2.1 Level AA. Partially conformant means that some parts of the content do not fully conform to the accessibility standard.

Accessibility Features

We have implemented the following accessibility features across our website:

  • Skip to main content link for keyboard navigation
  • Semantic HTML5 structure with proper heading hierarchy
  • ARIA labels on interactive elements and navigation
  • High color contrast ratios meeting WCAG AA standards (4.5:1 for normal text, 3:1 for large text)
  • Visible focus indicators for keyboard users
  • Descriptive alt text for images
  • Responsive design that works across all viewport sizes
  • Structured data and breadcrumb navigation
  • Multiple language support with proper lang attributes
  • Mobile-friendly navigation with accessible menu

Technical Specifications

Accessibility of Hack23.com relies on the following technologies to work with your web browser and any assistive technologies or plugins installed on your computer:

  • HTML5 - Semantic markup and document structure
  • CSS3 - Visual styling with accessibility considerations
  • WAI-ARIA - Accessible Rich Internet Applications for enhanced accessibility
  • JavaScript - For interactive features (progressive enhancement)

These technologies are relied upon for conformance with the accessibility standards used.

Assessment Approach

Hack23 AB assessed the accessibility of Hack23.com by the following approaches:

  • Self-evaluation with automated tools (HTMLHint, Lighthouse, axe DevTools)
  • Manual testing of keyboard navigation
  • Code review for semantic HTML and ARIA implementation
  • Color contrast analysis using WebAIM Contrast Checker
  • Cross-browser testing across Chrome, Firefox, Safari, and Edge

Automated Testing

We use the following automated tools in our CI/CD pipeline:

  • Lighthouse accessibility audits on every deployment
  • HTMLHint for HTML validation
  • ZAP Security Scanner which includes accessibility checks

Known Limitations

Despite our best efforts to ensure accessibility of Hack23.com, there may be some limitations. Below is a description of known limitations:

Potential Limitations

  • Third-party content: External badges and shields (e.g., GitHub badges) may not fully conform to accessibility standards as they are loaded from third-party services.
  • PDF documents: Some PDF documents may not be fully accessible. We are working to improve PDF accessibility.
  • Complex data visualizations: Some technical diagrams may require alternative text descriptions which we are continuously improving.

Compliance with Regulations

This accessibility statement applies to hack23.com and all its subpages.

EU Web Accessibility Directive

While Hack23 AB is a private company and not directly subject to the EU Web Accessibility Directive, we voluntarily comply with its requirements as part of our commitment to digital accessibility and inclusive design.

Swedish Accessibility Requirements

As a Swedish company, we are aware of and strive to comply with Swedish accessibility legislation, including the requirements for digital services outlined in Swedish law.
