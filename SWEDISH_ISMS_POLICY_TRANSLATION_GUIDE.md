## Translation Workflow (6 Phases)

### Phase 1: Preparation
1. **Copy English source file**
   ```bash
   cp discordian-[name].html discordian-[name]-policy_sv.html
   ```
2. **Update HTML lang attribute**
   ```html
   <html lang="sv">
   ```
3. **Add all hreflang tags** (minimum 13 tags)
4. **Set og:locale** to `sv_SE`

### Phase 2: Metadata Translation
1. **Title tag** - Translate with Swedish keywords
2. **Meta description** - Professional Swedish summary
3. **Meta keywords** - Swedish + English technical terms
4. **og:title, og:description** - Localized social metadata

### Phase 3: Schema.org Localization
1. **Set `inLanguage: "sv"`**
2. **Translate headline and description**
3. **Update breadcrumb names:**
   - Home ‚Üí Hem
   - Blog ‚Üí Blogg
4. **Keep author names in English** (Hagbard Celine, Simon Moon, etc.)

### Phase 4: Content Translation
1. **Headers and titles** - Professional Swedish
2. **Body paragraphs** - Maintain technical accuracy
3. **Lists and tables** - Translate headers, keep technical terms where appropriate
4. **Code examples** - Keep in English
5. **Preserve Discordian voice:**
   - "T√§nk sj√§lv" instead of "Think for yourself"
   - "FNORD" stays as-is
   - "23 FNORD 5" signature preserved

### Phase 5: Navigation & Footer
1. **Breadcrumbs** - Swedish navigation (Hem, Blogg, etc.)
2. **Header menu** - Link to Swedish versions
3. **Footer links** - Update to Swedish pages where available
4. **Related policies** - Link to Swedish versions with `(EN)` indicator for English-only

### Phase 6: Quality Validation
1. **HTML validation** - W3C validator
