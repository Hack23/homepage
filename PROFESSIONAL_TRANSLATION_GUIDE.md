## Translation Workflow

### Step 1: Setup and Review
1. **Read English source** thoroughly
2. **Review existing metadata** in target file (already translated)
3. **Identify terminology** needing specialized translation
4. **Note cultural references** requiring adaptation
5. **Review translation guides** for your language:
   - `German-Translation-Guide.md`
   - `Spanish-Translation-Guide.md`
   - `French-Translation-Guide.md`
   - `Dutch-Translation-Guide.md`

### Step 2: Translation Process
1. **Replace translation notice section** with translated content
2. **Translate section by section** maintaining HTML structure
3. **Keep HTML tags unchanged** (e.g., `<h2>`, `<p>`, `<strong>`)
4. **Preserve links** but translate link text
5. **Adapt examples** for cultural relevance
6. **Maintain Discordian voice** while ensuring clarity

### Step 3: Quality Assurance
1. **Technical accuracy check:** Verify cybersecurity terminology
2. **Readability test:** Ensure business-appropriate tone
3. **Link validation:** Check all internal/external links work
4. **HTML validation:** Ensure no broken tags
5. **Consistency check:** Compare with existing translations
6. **Native speaker review:** Get colleague to review if possible

### Step 4: Final Review
1. **Discordian voice preserved?** Counter-cultural tone maintained?
2. **Technical precision?** Terms accurate for security professionals?
3. **Business register?** Appropriate for C-suite audience?
4. **Cultural adaptation?** References make sense for local market?
5. **Code examples intact?** No translation of technical code?

## Common Pitfalls to Avoid

### ‚ùå Don't Do This:

1. **Literal Translation:** "Security through obscurity" ‚Üí Don't translate word-for-word
2. **Lost Nuance:** Discordian humor requires cultural adaptation, not direct translation
3. **Technical Errors:** Confusing "encryption" with "encoding" in translations
4. **Broken HTML:** Removing or changing HTML tags accidentally
5. **Inconsistency:** Using different terms for same concept within document
6. **Over-simplification:** Dumbing down technical content for general audience
7. **Lost Links:** Translating URLs or breaking anchor links
8. **Cultural Mismatch:** References that don't make sense in target market

### ‚úÖ Do This Instead:

1. **Conceptual Translation:** Convey the meaning, not just words
2. **Adapt Humor:** Make it work in target culture while preserving spirit
3. **Technical Precision:** Use exact industry-standard terminology
4. **Preserve Structure:** Keep all HTML formatting intact
5. **Term Consistency:** Use glossaries to maintain terminology standards
6. **Maintain Depth:** Keep technical sophistication appropriate for audience
7. **Link Validation:** Test all links after translation
8. **Cultural Relevance:** Adapt examples and references for local context
