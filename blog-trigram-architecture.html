Unity Architecture: The Five-Layer System

Unity game engine = flexible. Too flexible. Without architectural discipline, becomes spaghetti. We imposed five-layer structure:

1. üéÆ Presentation Layer: Unity Canvas

UI components, health bars, stamina meters, round timer, victory conditions. Five UI panels: HUD, pause menu, settings, character select, results screen.

2. üé¨ Animation Layer: Mecanim

Fighter animations, blend trees, transitions, inverse kinematics. Five animation states per fighter: idle, strike, grapple, defend, stunned. 5√ó5 = 25 total states.

3. ‚öôÔ∏è Logic Layer: Combat System

4. üéØ Physics Layer: Unity Physics

Collision detection, ragdoll physics, force application, ground detection. Five collider types: hitbox, hurtbox, pushbox, grabbox, groundbox.

5. üíæ Data Layer: ScriptableObjects

Unity projects without layer enforcement become unmaintainable. Our five-layer separation enables confident refactoring‚Äîchange UI without touching physics, modify AI without breaking animations.

  4. Cultural Consultation: Korean cultural advisors reviewing content. 5 review checkpoints: concept, design, implementation, testing, launch. Ensuring authenticity, avoiding appropriation.
  5. Open Source Education: Game assets documented in ART_ASSETS.md and AUDIO_ASSETS.md. Students studying Korean martial arts can examine technique implementation. Knowledge sharing through transparency.
