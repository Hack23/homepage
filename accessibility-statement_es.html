♿ Declaración de Accesibilidad para Hack23.com

Nuestro compromiso con la accesibilidad digital y la conformidad WCAG 2.1 Level AA

🏠 Inicio 📖 Blog de Seguridad Contáctenos

Introducción

Hack23 AB se compromete a garantizar la accesibilidad digital para personas con discapacidad. Mejoramos continuamente la experiencia del usuario para todos y aplicamos los estándares de accesibilidad pertinentes.

Estado de conformidad

Las Pautas de Accesibilidad para el Contenido Web (WCAG) definen requisitos para diseñadores y desarrolladores con el fin de mejorar la accesibilidad para personas con discapacidad. Definen tres niveles de conformidad: Level A, Level AA y Level AAA.

Hack23.com es parcialmente conforme con WCAG 2.1 Level AA. Parcialmente conforme significa que algunas partes del contenido no cumplen completamente con el estándar de accesibilidad.

Características de accesibilidad

Hemos implementado las siguientes características de accesibilidad en nuestro sitio web:

  • Enlace "Ir al contenido principal" para navegación por teclado
  • Estructura HTML5 semántica con jerarquía de encabezados adecuada
  • Etiquetas ARIA en elementos interactivos y navegación
  • Ratios de alto contraste de color que cumplen con los estándares WCAG AA (4.5:1 para texto normal, 3:1 para texto grande)
  • Indicadores de foco visibles para usuarios de teclado
  • Texto alternativo descriptivo para imágenes
  • Diseño responsive que funciona en todos los tamaños de viewport
  • Datos estructurados y navegación por migas de pan
  • Soporte multilingüe con atributos lang apropiados
  • Navegación mobile-friendly con menú accesible

Especificaciones técnicas

La accesibilidad de Hack23.com depende de las siguientes tecnologías para funcionar con su navegador web y cualquier tecnología de asistencia o plugins instalados en su ordenador:

  • HTML5 - Marcado semántico y estructura del documento
  • CSS3 - Estilos visuales con consideraciones de accesibilidad
  • WAI-ARIA - Aplicaciones de Internet Enriquecidas Accesibles para accesibilidad mejorada
  • JavaScript - Para funciones interactivas (mejora progresiva)

Estas tecnologías son necesarias para la conformidad con los estándares de accesibilidad utilizados.

Enfoque de evaluación

Hack23 AB evaluó la accesibilidad de Hack23.com mediante los siguientes enfoques:

  • Autoevaluación con herramientas automatizadas (HTMLHint, Lighthouse, axe DevTools)
  • Pruebas manuales de navegación por teclado
  • Revisión de código para HTML semántico e implementación de ARIA
  • Análisis de contraste de color usando WebAIM Contrast Checker
  • Pruebas multi-navegador en Chrome, Firefox, Safari y Edge

Pruebas automatizadas

Utilizamos las siguientes herramientas automatizadas en nuestro pipeline CI/CD:

  • Lighthouse auditorías de accesibilidad en cada despliegue
  • HTMLHint para validación HTML
  • ZAP Security Scanner que incluye verificaciones de accesibilidad

Limitaciones conocidas

A pesar de nuestros mejores esfuerzos para garantizar la accesibilidad de Hack23.com, pueden existir algunas limitaciones. A continuación se describe las limitaciones conocidas:

Limitaciones potenciales

  • Contenido de terceros: Las insignias y shields externos (por ej. insignias de GitHub) pueden no cumplir completamente con los estándares de accesibilidad ya que se cargan desde servicios de terceros.
  • Documentos PDF: Algunos documentos PDF pueden no ser completamente accesibles. Estamos trabajando para mejorar la accesibilidad de los PDF.
  • Visualizaciones de datos complejas: Algunos diagramas técnicos pueden requerir descripciones textuales alternativas que estamos mejorando continuamente.

Cumplimiento de normativas

Esta declaración de accesibilidad se aplica a hack23.com y todas sus subpáginas.

Directiva Europea de Accesibilidad Web

Aunque Hack23 AB es una empresa privada y no está directamente sujeta a la Directiva Europea de Accesibilidad Web, cumplimos voluntariamente con sus requisitos como parte de nuestro compromiso con la accesibilidad digital y el diseño inclusivo.

Requisitos españoles de accesibilidad

Como empresa que opera en España, somos conscientes de y nos esforzamos por cumplir con la legislación española sobre accesibilidad, incluidos los requisitos para servicios digitales establecidos en la ley española.
