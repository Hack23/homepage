♿ Tilgængelighedserklæring for Hack23.com

Vores forpligtelse til digital tilgængelighed og WCAG 2.1 niveau AA compliance

🏠 Hjem 📖 Sikkerhedsblog Kontakt Os

Introduktion

Hack23 AB er forpligtet til at sikre digital tilgængelighed for personer med handicap. Vi forbedrer løbende brugeroplevelsen for alle og anvender relevante tilgængelighedsstandarder.

Overensstemmelsesstatus

Web Content Accessibility Guidelines (WCAG) definerer krav til designere og udviklere for at forbedre tilgængeligheden for personer med handicap. Det definerer tre overensstemmelsesniveauer: Niveau A, Niveau AA og Niveau AAA.

Hack23.com er delvist overensstemmende med WCAG 2.1 Niveau AA. Delvist overensstemmende betyder, at nogle dele af indholdet ikke fuldt ud overholder tilgængelighedsstandarden.

Tilgængelighedsfunktioner

Vi har implementeret følgende tilgængelighedsfunktioner på tværs af vores hjemmeside:

  • Spring til hovedindhold link til tastaturnavigation
  • Semantisk HTML5 struktur med korrekt overskriftshierarki
  • ARIA-mærkater på interaktive elementer og navigation
  • Høj farvekontrastforhold der opfylder WCAG AA-standarder (4,5:1 for normal tekst, 3:1 for stor tekst)
  • Synlige fokusindikatorer for tastaturbrugere
  • Beskrivende alt-tekst til billeder
  • Responsivt design der fungerer på tværs af alle viewport-størrelser
  • Strukturerede data og brødkrummenavigation
  • Understøttelse af flere sprog med korrekte lang-attributter
  • Mobilvennlig navigation med tilgængeligt menu

Tekniske Specifikationer

Tilgængeligheden af Hack23.com er afhængig af følgende teknologier for at fungere med din webbrowser og eventuelle hjælpeteknologier eller plugins installeret på din computer:

  • HTML5 - Semantisk markering og dokumentstruktur
  • CSS3 - Visuel styling med tilgængelighedsovervejelser
  • WAI-ARIA - Accessible Rich Internet Applications til forbedret tilgængelighed
  • JavaScript - Til interaktive funktioner (progressiv forbedring)

Disse teknologier er nødvendige for overensstemmelse med de anvendte tilgængelighedsstandarder.

Vurderingsmetode

Hack23 AB vurderede tilgængeligheden af Hack23.com ved følgende metoder:

  • Selvevaluering med automatiserede værktøjer (HTMLHint, Lighthouse, axe DevTools)
  • Manuel testning af tastaturnavigation
  • Kodegennemgang for semantisk HTML og ARIA-implementering
  • Farvekontrastanalyse ved brug af WebAIM Contrast Checker
  • Tværbrowser-testning på tværs af Chrome, Firefox, Safari og Edge

Automatiseret Testning

Vi bruger følgende automatiserede værktøjer i vores CI/CD-pipeline:

  • Lighthouse tilgængelighedsaudits ved hver deployment
  • HTMLHint til HTML-validering
  • ZAP Security Scanner som inkluderer tilgængelighedstjek

Kendte Begrænsninger

På trods af vores bedste bestræbelser på at sikre tilgængelighed af Hack23.com, kan der være nogle begrænsninger. Nedenfor er en beskrivelse af kendte begrænsninger:

Potentielle Begrænsninger

  • Tredjeparts indhold: Eksterne badges og shields (f.eks. GitHub badges) overholder muligvis ikke fuldt ud tilgængelighedsstandarder, da de indlæses fra tredjepartstjenester.
  • PDF-dokumenter: Nogle PDF-dokumenter er muligvis ikke fuldt tilgængelige. Vi arbejder på at forbedre PDF-tilgængelighed.
  • Komplekse datavisualiseringer: Nogle tekniske diagrammer kan kræve alternative tekstbeskrivelser, som vi løbende forbedrer.

Overholdelse af Regler

Denne tilgængelighedserklæring gælder for hack23.com og alle dens undersider.

EU Webtilgængelighedsdirektiv

Selvom Hack23 AB er en privat virksomhed og ikke direkte er underlagt EU Webtilgængelighedsdirektivet, overholder vi frivilligt dets krav som en del af vores forpligtelse til digital tilgængelighed og inkluderende design.

Danske Tilgængelighedskrav

Som en virksomhed der betjener Danmark, er vi opmærksomme på og stræber efter at overholde dansk tilgængelighedslovgivning, herunder kravene til digitale tjenester beskrevet i dansk lov.
