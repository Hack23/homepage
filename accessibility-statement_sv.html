♿ Tillgänglighetsredogörelse för Hack23.com

Vårt engagemang för digital tillgänglighet och WCAG 2.1 Nivå AA-efterlevnad

🏠 Hem 📖 Säkerhetsblogg Kontakta oss

Introduktion

Hack23 AB strävar efter att säkerställa digital tillgänglighet för personer med funktionsnedsättningar. Vi förbättrar kontinuerligt användarupplevelsen för alla och tillämpar relevanta tillgänglighetsstandarder.

Överensstämmelsestatus

Web Content Accessibility Guidelines (WCAG) definierar krav för designers och utvecklare för att förbättra tillgängligheten för personer med funktionsnedsättningar. Det definierar tre nivåer av överensstämmelse: Nivå A, Nivå AA och Nivå AAA.

Hack23.com är delvis överensstämmande med WCAG 2.1 Nivå AA. Delvis överensstämmande innebär att vissa delar av innehållet inte helt uppfyller tillgänglighetsstandarderna.

Tillgänglighetsfunktioner

Vi har implementerat följande tillgänglighetsfunktioner på vår webbplats:

  • Hoppa till huvudinnehåll-länk för tangentbordsnavigering
  • Semantisk HTML5-struktur med korrekt rubrikhierarki
  • ARIA-etiketter på interaktiva element och navigering
  • Hög färgkontrast som uppfyller WCAG AA-standarder (4.5:1 för normal text, 3:1 för stor text)
  • Synliga fokusindikatorer för tangentbordsanvändare
  • Beskrivande alt-text för bilder
  • Responsiv design som fungerar i alla visningsstorlekar
  • Strukturerad data och brödsmulenavigering
  • Stöd för flera språk med korrekta lang-attribut
  • Mobilvänlig navigering med tillgänglig meny

Tekniska specifikationer

Tillgängligheten för Hack23.com bygger på följande tekniker för att fungera med din webbläsare och eventuella hjälpmedel eller tillägg installerade på din dator:

  • HTML5 - Semantisk uppmärkning och dokumentstruktur
  • CSS3 - Visuell styling med tillgänglighetshänsyn
  • WAI-ARIA - Accessible Rich Internet Applications för förbättrad tillgänglighet
  • JavaScript - För interaktiva funktioner (progressiv förbättring)

Dessa tekniker används för att uppfylla de använda tillgänglighetsstandarderna.

Utvärderingsmetod

Hack23 AB har utvärderat tillgängligheten för Hack23.com med följande metoder:

  • Självutvärdering med automatiserade verktyg (HTMLHint, Lighthouse, axe DevTools)
  • Manuell testning av tangentbordsnavigering
  • Kodgranskning för semantisk HTML och ARIA-implementering
  • Färgkontrastanalys med WebAIM Contrast Checker
  • Testning i flera webbläsare över Chrome, Firefox, Safari och Edge

Automatiserad testning

Vi använder följande automatiserade verktyg i vår CI/CD-pipeline:

  • Lighthouse tillgänglighetsgranskningar vid varje distribution
  • HTMLHint för HTML-validering
  • ZAP Security Scanner som inkluderar tillgänglighetskontroller

Kända begränsningar

Trots våra bästa ansträngningar att säkerställa tillgängligheten för Hack23.com kan det finnas vissa begränsningar. Nedan följer en beskrivning av kända begränsningar:

Potentiella begränsningar

  • Tredjepartsinnehåll: Externa märken och sköldar (t.ex. GitHub-märken) kanske inte helt uppfyller tillgänglighetsstandarder eftersom de laddas från tredjepartstjänster.
  • PDF-dokument: Vissa PDF-dokument är kanske inte helt tillgängliga. Vi arbetar med att förbättra PDF-tillgängligheten.
  • Komplexa datavisualiseringar: Vissa tekniska diagram kan kräva alternativa textbeskrivningar som vi kontinuerligt förbättrar.

Efterlevnad av regelverk

Denna tillgänglighetsredogörelse gäller hack23.com och alla dess undersidor.

EU:s webbtillgänglighetsdirektiv

Även om Hack23 AB är ett privat företag och inte direkt omfattas av EU:s webbtillgänglighetsdirektiv, följer vi frivilligt dess krav som en del av vårt engagemang för digital tillgänglighet och inkluderande design.

Svenska tillgänglighetskrav

Som ett svenskt företag är vi medvetna om och strävar efter att följa svensk tillgänglighetslagstiftning, inklusive kraven för digitala tjänster som anges i svensk lag.
