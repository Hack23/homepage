♿ Déclaration d'accessibilité pour Hack23.com

Notre engagement envers l'accessibilité numérique et la conformité WCAG 2.1 Level AA

🏠 Accueil 📖 Blog Sécurité Nous contacter

Introduction

Hack23 AB s'engage à garantir l'accessibilité numérique pour les personnes en situation de handicap. Nous améliorons continuellement l'expérience utilisateur pour tous et appliquons les normes d'accessibilité pertinentes.

État de conformité

Les Web Content Accessibility Guidelines (WCAG) définissent les exigences pour les concepteurs et les développeurs afin d'améliorer l'accessibilité pour les personnes en situation de handicap. Elles définissent trois niveaux de conformité : Level A, Level AA et Level AAA.

Hack23.com est partiellement conforme avec WCAG 2.1 Level AA. Partiellement conforme signifie que certaines parties du contenu ne sont pas entièrement conformes à la norme d'accessibilité.

Fonctionnalités d'accessibilité

Nous avons implémenté les fonctionnalités d'accessibilité suivantes sur notre site web :

  • Lien "Aller au contenu principal" pour la navigation au clavier
  • Structure HTML5 sémantique avec hiérarchie de titres appropriée
  • Labels ARIA sur les éléments interactifs et la navigation
  • Rapports de contraste des couleurs élevés conformes aux normes WCAG AA (4,5:1 pour le texte normal, 3:1 pour le texte large)
  • Indicateurs de focus visibles pour les utilisateurs de clavier
  • Texte alternatif descriptif pour les images
  • Design responsive fonctionnant sur toutes les tailles de viewport
  • Données structurées et navigation par fil d'Ariane
  • Support multilingue avec attributs lang appropriés
  • Navigation mobile-friendly avec menu accessible

Spécifications techniques

L'accessibilité de Hack23.com repose sur les technologies suivantes pour fonctionner avec votre navigateur web et les technologies d'assistance ou plugins installés sur votre ordinateur :

  • HTML5 - Balisage sémantique et structure du document
  • CSS3 - Styles visuels avec considérations d'accessibilité
  • WAI-ARIA - Accessible Rich Internet Applications pour une accessibilité améliorée
  • JavaScript - Pour les fonctionnalités interactives (amélioration progressive)

Ces technologies sont nécessaires pour la conformité avec les normes d'accessibilité utilisées.

Approche d'évaluation

Hack23 AB a évalué l'accessibilité de Hack23.com selon les approches suivantes :

  • Auto-évaluation avec des outils automatisés (HTMLHint, Lighthouse, axe DevTools)
  • Tests manuels de la navigation au clavier
  • Revue de code pour le HTML sémantique et l'implémentation ARIA
  • Analyse du contraste des couleurs avec WebAIM Contrast Checker
  • Tests cross-navigateurs sur Chrome, Firefox, Safari et Edge

Tests automatisés

Nous utilisons les outils automatisés suivants dans notre pipeline CI/CD :

  • Lighthouse audits d'accessibilité à chaque déploiement
  • HTMLHint pour la validation HTML
  • ZAP Security Scanner qui inclut des vérifications d'accessibilité

Limitations connues

Malgré nos meilleurs efforts pour garantir l'accessibilité de Hack23.com, certaines limitations peuvent exister. Voici une description des limitations connues :

Limitations potentielles

  • Contenu tiers : Les badges et shields externes (par ex. badges GitHub) peuvent ne pas être entièrement conformes aux normes d'accessibilité car ils sont chargés depuis des services tiers.
  • Documents PDF : Certains documents PDF peuvent ne pas être entièrement accessibles. Nous travaillons à améliorer l'accessibilité des PDF.
  • Visualisations de données complexes : Certains diagrammes techniques peuvent nécessiter des descriptions textuelles alternatives que nous améliorons continuellement.

Conformité aux réglementations

Cette déclaration d'accessibilité s'applique à hack23.com et toutes ses sous-pages.

Directive européenne sur l'accessibilité du web

Bien que Hack23 AB soit une entreprise privée et non directement soumise à la Directive européenne sur l'accessibilité du web, nous nous conformons volontairement à ses exigences dans le cadre de notre engagement envers l'accessibilité numérique et le design inclusif.

Exigences françaises en matière d'accessibilité

En tant qu'entreprise opérant en France, nous sommes conscients de et nous efforçons de nous conformer à la législation française sur l'accessibilité, y compris les exigences pour les services numériques définies par la loi française.
