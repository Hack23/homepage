Svensk översättning under utveckling

Denna sida är under utveckling för svensk översättning. Innehållet kommer att översättas med bevarande av Discordian-rösten och professionell svensk ISO 27001-terminologi.

🔑 Åtkomstkontrollpolicy – Översikt:

  • Zero Trust-arkitektur: AWS Identity Center SSO med centraliserad autentisering
  • MFA 100%: Multifaktorautentisering på alla system utan undantag
  • 90-dagars granskningar: Automatisk detektering av vilande konton
  • Klassificeringsdriven åtkomst: Behörigheter baserade på dataklassificering
  • Semi-årliga policy-översyner: Levande dokumentation som anpassas efter hot

För fullständigt innehåll, se:

  • 📄 Engelska versionen — Fullständigt innehåll med Discordian-filosofi
  • 📋 Offentlig ISMS-policy — Tekniska detaljer på GitHub
  • 🔒 Relaterad åtkomstkontroll — Övergripande åtkomstkontroll

UPPLYSNING: Åtkomstkontroll handlar inte om tillit — det handlar om verifiering. AWS Identity Center + obligatorisk MFA = systematisk verifiering i skala. Välj dokumenterade kontroller framför säkerhet genom hopp.

Nyckelterminologi:

  • Åtkomstkontroll: Access Control
  • Zero Trust: Zero Trust / Nolltillit
  • Multifaktorautentisering (MFA): Multi-Factor Authentication
  • Minsta behörighet: Least Privilege / Principle of Least Privilege
  • Identitetscentrerad säkerhet: Identity-Centric Security
  • Vilande konton: Dormant Accounts
  • Åtkomstmatris: Access Matrix
  • Privilegierad åtkomst: Privileged Access
